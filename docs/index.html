How it works

Instead of loading multi-kilobyte markdown files on every session, OpenClaw reads a single-line Cypher directive ‚Äî then queries an embedded graph database to build structured, prioritized context on demand.

1

Workspace files become stubs

SOUL.md, MEMORY.md, TOOLS.md, AGENTS.md each contain one line ‚Äî a Cypher query directive. No content, no secrets, no growing files.

2

Three-layer cache ‚Äî correctly ordered

workspaceFileCache stores raw stubs (mtime). graphQueryCache owns resolved content (60s TTL). graphQueryInFlight deduplicates concurrent misses. Each layer does exactly one job.

3

Non-blocking async queries

Graph queries use execFileAsync instead of execSync. A slow or locked DB never stalls the event loop. Agents keep running while the query resolves.

4

Multi-instance, per-copy DB

Each OpenClaw instance gets its own local DB copy via install.sh. Sub-millisecond queries, no network dependency, no lock contention across agents.

Vector Embeddings ‚Äî Semantic Skill Search

All 322 skills ship with precomputed text-embedding-3-small vectors (1536 dimensions). Cosine similarity search finds semantically relevant skills ‚Äî not just keyword matches.

# Semantic search ‚Äî finds skills by meaning, not keywords
node ladybugdb/scripts/query.js --semantic "deploy containerized app to cloud"
# ‚Üí devops-sre/docker, cloud-aws/aws-ecs, cloud-gcp/gcp-cloud-run, ...

# Generate your own embeddings (OpenAI or local Ollama nomic-embed-text)
node ladybugdb/scripts/gen-embeddings.js --missing

Benchmarked: 322 skills embedded in 8.4s via OpenAI API ¬∑ 0 errors ¬∑ cost <$0.01

After the first workspace load per session, all results are served from an in-process JavaScript Map cache with a 60-second TTL. Subsequent prompt builds, tool calls, and compaction cycles cost 0ms for workspace content.

For multi-instance deployments (e.g., a fleet of 8 AI agents sharing one DB), LadybugDB supports concurrent reads without blocking. Total token savings across an 8-instance fleet: ~65,000+ tokens saved per hour.

Quick Start

Get running in 5 steps

1

Clone and install

git clone https://github.com/alphaonedev/openclaw-graph.git
cd openclaw-graph
npm install
2

Load the skill database

node ladybugdb/scripts/loader.js
# ‚Üí 322 skills | 28 clusters | 247 edges | 0 errors
3

Seed your workspace identity

Edit ladybugdb/scripts/seed-workspace.js ‚Äî set your WORKSPACE_ID and fill in Soul, Memory, Tool nodes. Then:

node ladybugdb/scripts/seed-workspace.js
# ‚Üí Soul: 5  Memory: 3  Tool: 8  Skill: 322
4

Patch OpenClaw workspace.ts

Add the graph backend block from src/agents/workspace.graph.ts into your OpenClaw's src/agents/workspace.ts. See the README for the exact patch points.

cd path/to/openclaw
pnpm build   # recompile after patching
5

Deploy workspace stubs

cp workspace-stubs/SOUL.md    ~/.openclaw/workspace/SOUL.md
cp workspace-stubs/MEMORY.md  ~/.openclaw/workspace/MEMORY.md
cp workspace-stubs/TOOLS.md   ~/.openclaw/workspace/TOOLS.md
cp workspace-stubs/AGENTS.md  ~/.openclaw/workspace/AGENTS.md
# Update 'myapp' in each file to your WORKSPACE_ID
